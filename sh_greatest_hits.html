 Accessing Return and Exit Values of Commands
 --------------------------------------------

 The ` ` operators work fine in many cases, but this doesn't:

   echo `basename `dirname `pwd```

 This does:

   echo $(basename $(dirname $(pwd)))

 Numeric exit values are stored in "$?". So:

   <command>; echo $?

 gives the exit value.

 Checking File for Special Characters
 --------------------------------------

 Sometimes a file that is processed contains special characters
 that cause problems for shell commands. This can be used to check
 for such characters:

   # flag non-ASCII, punctuation, and control sequences
   grep -En --color=auto "[^[:ascii:]]|[[:punct:][:control:]]" <file>

 This can also be used for strings in a pipeline or to check your
 shell script. In the latter case, dropping [:punct:] would be desirable.

   # general check for punctuation chararcters

   then
     echo -e "\n  ERROR: \"$2\" cannot have special characters.\n";
     exit 1;
   fi;

 ### more sophisticated processing
   #! /bin/sh

 # used to process command line args of the form "-x".

   USAGE="Usage: $(basename $0) [-hv] [-o arg] args";

 # Parse command line options.
 # the colon means the option takes an argument
   while getopts hvo: OPT; do
     case "$OPT" in
        h)
          echo $USAGE;
 	  exit 0;
 	  ;;
 	v)
 	  echo "$(basename $0) version 0.1";
 	  exit 0;
 	  ;;
 	o)
 	  OUTPUT_FILE=$OPTARG;
 	  ;;
 	\?)
 # getopts issues an error message
 	  echo $USAGE >&2;
 	  exit 1;
 	  ;;
 	esac;
   done;

 # Remove the switches we parsed above.
 # This only works in a POSIX compliant shell.
   shift $(($OPTIND -  1));

 # We want at least one non-option argument.
 # Remove this block if you don't need it.
   if [ $# -eq 0 ]; then
     echo $USAGE >&2;
     exit 1;
   fi;

 # Access additional arguments as usual through
 # variables $@, $*, $1, $2, etc. or using this loop:
   for PARAM; do
 # when the 'in' keyword is omitted, the command line args are
 # iterated over.
 # as the remaining arguments are iterated over, you would probably
 # use some if blocks here to test what those are and handle them
 # appropriately.
     echo $PARAM;
   done;

   echo "\$1 is $1";

   exit 0;

 Commands Having Syntactical Elements Resolved Through Variables
 ---------------------------------------------------------------

 In cases where a variables are used as a part of a command's syntax,
 it can be necessary to use 'eval' to get an additional round of parsing.

   eval dname_$i=`$BASENAME "$dfile"`;

   $CAT "${projectDir}/${p}/$file" | eval $SED "$draftCmd" \
   "$draftCondCmd" "$draftDateCmd" "$versionCmd" "$csdSearchCmd" \
   "$copyrightCmd"  > ${projectDir}/${p}/${file}.1;

   eval $ASPELL -l en create master ./\$dname_$i.tmpdict~ < "$dfile";

 This is typically discovered on a case-by-case basis.

 Escaping a Backslash in a Regular Expression
 --------------------------------------------

 Escaping backslashes can be a bit tricky and depends on context.

 Sometimes it's necessary to replace "/" characters with "\/" to ensure
 proper behavior in a pipeline or some downstream command. Here is an
 example sed expression to do this:

   's#/#\\\/#g'

 The initial \\ ensures \ is maintained, and \/ ensures \ is maintained in
 the environment that process the output.



 Escaping Special Characters in XML to Create a sed Command
 ----------------------------------------------------------

   # escape /, ;, and " in a string.
   # notice that three \\\ characters are required for an escape.
   oldTitle="`cat "/tmp/$fileout" | awk '/<title/,/title>/' | sed -e 's#\/#\\\/#g' -e 's#\;#\\\;#g' -e 's#\"#\\\"#g'`";
   sedCommand=' -e "s/$oldTitle/<title>$title<\/title>/"';

 Matching Text With Line Breaks
 ------------------------------

   awk -v n=1 '/END/{p=0}; p && c == n; /BEGIN/ && !p {p=1; c++}' <file>

 # the n variable defines which match gets printed. in this case, the
 # first. the END and BEGIN strings define the boundaries of the match.
 # when the ending string is matched, p is set to 0, which turns
 # printing off.
 # if "p & & c" is equal to n, the record is printed.
 # if the starting string is matched and p=0 or null, p is set to 1
 # and the match count is incremented.
 # special characters need to be escaped. in csh, the ! needs to be
 # escaped.

 Sorting Lines With Similar Strings
 ----------------------------------

 The Awk script used here works in gawk. It may not work in other
 Awk versions.

   cat log | gawk 'match($0, /REAL TIME = [0-9]+/, arr) { print arr[0] ": " $0 }' | sed 's/^REAL TIME =//' | sort -t ":" -k 1 -nr

 This matches the longest, leftmost occurrance of "REAL TIME = n" on a
 line, then prints the line with a "n: " prefix. Lines that don't
 match the pattern aren't printed. These lines are then sorted by the
 prefix.

 Spell Checking a Text File Created by pdftotext
 -----------------------------------------------

 The main interest here is the pre-processing of the input file, then
 setting up a command sequence using printf to pipe into the shell,
 which in this case is dash. The $SPELL variable pointed to the
 'aspell' program command line.

 # main engine for producing a spelling_report file.
 # this routine permits page breaks in the input text file. page breaks
 # are represented by \014. if breaks are present, then spelling
 # discrepancies are written by page number.
 # \42 is the code for a double quote. this is needed so the echo
 # command passes the entire contents of the file to the spelling
 # program. in order to get quotes past the shell and into the subshell
 # they need to be escaped in the sh script.
 # double quotes need to be stripped from the input file, otherwise
 # they get nested what cat-ing the input, which gives errors in
 # the echo command in the awk script. additionally, certain sequences
 # of $ followed by other characters get interpreted by the shell
 # script and funky things happen. for example $_ causes an odd
 # issue. so $ needs to be changed to something else. another problem-
 # atic character is ` because that means shell command substitution.
 # hence, we change `, ", and $ to |.
 # pdftotext can create odd characters that include ^ symbols.
 # these strange characters are stripped using sed.
   check() {
     cat $inFile | sed 's/[\"\$\`]/\|/g' | sed 's/[âÂ]//g'|  awk -v SPELL="$SPELL" -F"\014" '
       BEGIN {RS="\014"; print "\nSpelling Report\n"; print "Aspell command: "SPELL" list"; print "---------------"} {
         n=NR;
         printf("list=`echo %s | %s list`; \
         page=%s; \
           if [ \"$list\" != \"\" ]; then \
             touch ./spellchk~; \
             echo \"\"; \
             echo \"Page: ${page}\"; \
             echo \"\"; \
             echo \"$list\"; \
             echo \"---------------\"; \
           fi;", "\42"$1"\42", SPELL, n) | "dash"
         close( "dash" );
       }' >> $outFile;
     if [ ! -f spellchk~ ]; then
       echo "No errors." >> "$outFile";
       echo "---------------" >> "$outFile";
     fi;
     rm spellchk~ 2> /dev/null;
   };

 XML Verification
 ----------------

 # write beautified XML
 $CAT "$t" | $DOS2UNIX | $XMLLINT --format -  | $SED -r 's%^[[:blank:]]+<%<%g' | \
   $AWK 'BEGIN {ORS = "@|@"} {print}' | $SED -e 's/>@|@\n "$tempFile";

 This takes input from a file, removes DOS end of lines, beautifies the
 XML, removes leading whitespace on a line, changes \n to @|@, and
 changes >@|@< to >\n<.
 This is to ensure appropriate newlines occur between element tags.
 After changing >@|@< to a newline, there are three remaining cases
 for where @|@ appears: between two graphical characters, between two
 space characters, or between a graph and a space character. These need
 special consideration because if the \n are present for formatting the
 original text, then you have to restore them. Otherwise, you can remove
 them.
 In the first case, we keep the graphical characters and insert a space
 between them. In the second two cases we simply drop the @|@. The result
 is left-justfied XML with tags starting and ending every line.

 # sanity check
 $XMLLINT --noout $tempFile;
 if [ $? -ne 0 ]; then $PRINTF "\n   INTERNAL ERROR: Wrote bad ${tempFile}, contact danl.\n"; exit 1; fi;
